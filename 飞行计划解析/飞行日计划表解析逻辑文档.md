# 飞行日计划表解析逻辑文档

## 1. 概述

本程序用于解析 Excel 格式的飞行日计划表，自动提取每个架次的课目编号、起止时间、前后仓人员信息及飞行类型（单飞/带飞）。

---

## 2. Excel 表格结构

### 2.1 整体布局

```
行1-6   : 表头信息（批准人、日期、标题等）
行7     : 小时行（hour_time）—— 标注整点时间 10:00, 11:00 ... 17:00
行8     : 分钟行（min_time）—— 标注分钟值 00, 10, 20, 30, 40, 50
行9起   : 数据行 —— 左侧为代字代号表，右侧为飞行计划时间轴
```

### 2.2 关键标记列

| 标记 | 位置 | 含义 |
|------|------|------|
| `飞机` | 第7行某列 | 数据区起始标记（data_start_column） |
| `备注` | 第7行某列 | 数据区结束标记（data_end_column） |

### 2.3 时间轴区域

时间轴位于 `飞机` 列之后、`备注` 列之前。

```
第7行:  10:00          飞机    11:00                12:00 ...    备注
第8行:  00  10  20  30  40     00  10  20 ...  50   00 ...
        ↑                       ↑
        飞机前（不参与计算）      飞机后（数据区开始）
```

---

## 3. 解析步骤

### Step 1：定位边界列

从第7行逐列扫描：

- 找到内容为 `"飞机"` 的列 → `data_start_column`
- 找到内容为 `"备注"` 的列 → `data_end_column`

飞机列之前的时间不参与计算，只从飞机列之后开始。

### Step 2：构建时间映射（time_map）

**核心规则：每一列 = 5 分钟。**

第8行的分钟值以每10分钟标注一次（00, 10, 20, 30, 40, 50），每个标注占2列（合并单元格）。实际含义是：

```
有分钟值的列 → 该分钟值对应的时间
紧邻的下一列（无分钟值） → 上一个分钟值 + 5分钟
```

**构建流程：**

1. 向飞机列之前回溯，找到最近的小时值（如 `10:00`），作为初始小时 `current_hour`
2. 从 `data_start_column + 1` 开始逐列扫描到 `data_end_column`：
   - 遇到第7行有小时值（如 `11:00`）→ 更新 `current_hour`
   - 遇到第8行有分钟值（如 `20`）→ 记录 `time_map[列号] = current_hour:20`，保存 `last_base_min = 20`
   - 遇到第8行无分钟值 → 记录 `time_map[列号] = current_hour:(last_base_min + 5)`
   - 若 `+5` 后分钟 ≥ 60，则分钟归零、小时进位

**示例（飞机列后开始）：**

```
col28 → row8=30 → 10:30
col29 → row8=空 → 10:35  (30+5)
col30 → row8=40 → 10:40
col31 → row8=空 → 10:45  (40+5)
...
col34 → row7=11:00, row8=00 → 11:00
col35 → row8=空 → 11:05  (00+5)
...
col63 →           → 13:25
col89 →           → 15:35
```

### Step 3：构建代字代号表（codeCipherMap）

从第9行开始，读取左侧区域的代字/代号列组：

| 列组 | 代字列 | 代号列 |
|------|--------|--------|
| 第1组 | 第3列 | 第4列 |
| 第2组 | 第8列 | 第9列 |
| 第3组 | 第13列 | 第14列 |

逐行扫描，当代字和代号都非空时，存入有序字典：

```
codeCipherMap = {
    "侯": "095",
    "恒": "247",
    "田": "413",
    "昊": "465",
    "帅": "568"
}
```

保持读取顺序（OrderedDict）。

### Step 4：识别架次（逐行扫描数据区）

从第9行开始，在 `data_start_column + 1` 到 `data_end_column` 范围内逐单元格扫描。

#### 4.1 识别课目编号（起始标记）

使用正则匹配：`^(LLNR|FM|LLN)[\w-]*$`

匹配规则：
- 以 `LLNR`、`FM`、`LLN` 开头
- 后面可以拼接字母/数字/下划线，或 `-数字`
- 示例：`LLNR`、`LLN-1`、`FM-4`、`FMa`

**课目编号所在的列 = 起始列，对应的 time_map 值 = 起始时间。**

#### 4.2 识别结束标记（代字+代号）

从课目编号所在列向右扫描同一行，使用正则匹配：`^(汉字)(\d{3,4})$`

即：1个汉字 + 3~4位数字。

**严格验证：**
- 提取出的汉字（代字）必须存在于 `codeCipherMap` 中
- 提取出的数字（代号）必须与 `codeCipherMap[代字]` 完全一致
- 不匹配的跳过，继续向右扫描

**找到有效结束标记后：**
- 该标记所在列 = `end_col`（结束标记列）
- 结束时间 = `time_map[end_col - 1]`（结束标记列的前一列对应的时间）
- 该标记的内容 = 前仓人员（代字 + 代号）

#### 4.3 判断单飞 / 带飞

在结束标记所在列（`end_col`），检查**下一行**（`课目所在行 + 1`）的同列单元格：

```
if 下一行同列 匹配 "汉字+数字" 且 通过codeCipherMap严格验证:
    → 带飞（前仓 + 后仓）
    → 下一行的内容 = 后仓人员
else:
    → 单飞（仅前仓）
```

---

## 4. 输出字段

每个架次输出以下信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| 课目编号 | 课目名称 | `LLN-1` |
| 起始时间 | 课目所在列的时间 | `11:00` |
| 结束时间 | 结束标记列-1的时间 | `12:00` |
| 所在行 | Excel行号 | `第 13 行` |
| 起始列 | 课目所在列号及字母 | `第 34 列 (AH列)` |
| 结束列 | 结束标记所在列号及字母 | `第 47 列 (AU列)` |
| 前仓代字 | 前仓人员代字 | `田` |
| 前仓代号 | 前仓人员代号 | `413` |
| 后仓代字 | 后仓人员代字（单飞为空） | `昊` |
| 后仓代号 | 后仓人员代号（单飞为空） | `465` |
| 飞行类型 | 单飞 / 带飞 | `带飞 (前仓+后仓)` |

## 5. 完整解析流程图

```
开始
 │
 ├─ Step1: 扫描第7行 → 定位"飞机"列和"备注"列
 │
 ├─ Step2: 构建时间映射
 │    ├─ 回溯飞机列前，取初始小时
 │    └─ 从飞机列+1 逐列到备注列
 │         ├─ 有分钟值 → time_map[列] = 小时:分钟
 │         └─ 无分钟值 → time_map[列] = 小时:(上个分钟+5)
 │
 ├─ Step3: 构建代字代号表
 │    └─ 从第9行读取3组代字/代号列 → codeCipherMap
 │
 ├─ Step4: 逐行扫描数据区 (第9行起)
 │    └─ 对每个单元格:
 │         ├─ 匹配课目正则? → 否 → 跳过
 │         └─ 是 → 记录起始列/时间
 │              ├─ 同行向右找结束标记(代字+代号，严格验证)
 │              │    ├─ 找到 → 记录结束列, end_col-1→结束时间, 标记内容→前仓
 │              │    └─ 未找到 → 跳过该架次
 │              └─ 检查下一行同列
 │                   ├─ 有代字+代号 → 带飞, 内容→后仓
 │                   └─ 无 → 单飞
 │
 └─ 输出结果
```

## 6. 配置常量说明

| 常量 | 值 | 说明 |
|------|---|------|
| `START_ROW` | 9 | 数据区起始行 |
| `HOUR_TIME_ROW` | 7 | 小时所在行 |
| `MIN_TIME_ROW` | 8 | 分钟所在行 |
| `CODE_CIPHER_START_ROW` | 9 | 代字代号表起始行 |
| `CODE_CIPHER_GROUPS` | (3,4), (8,9), (13,14) | 三组代字/代号列位置 |
| `COURSE_PATTERN` | `^(LLNR\|FM\|LLN)[\w-]*$` | 课目编号正则 |
| `PERSON_PATTERN` | `^(汉字)(\d{3,4})$` | 人员标记正则（1汉字+3~4位数字） |

---

## 7. 边界与异常处理

| 场景 | 处理方式 |
|------|----------|
| 课目编号所在列无时间映射 | 起始时间显示"未知" |
| 同行向右未找到有效结束标记 | 整个架次跳过不输出 |
| 代字不在 codeCipherMap 中 | 不算有效结束标记，继续向右扫描 |
| 代字匹配但代号不匹配 | 不算有效结束标记，继续向右扫描 |
| 数据区连续3行为空 | 认为数据区结束，停止扫描 |
| 文件中无"飞机"或"备注"列 | 抛出异常并提示 |

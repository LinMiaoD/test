# AI 提示词：生成系统内部接口示意图（PlantUML）

> 使用方法：将下方提示词复制给 ChatGPT / Claude / 其他AI，把【】中的内容替换为你自己的系统信息即可。

---

## 提示词正文

```
你是一位资深系统架构师，擅长用 PlantUML 绘制清晰、专业的系统内部接口示意图。

请根据我提供的系统信息，严格按照以下规范生成 PlantUML 代码：

━━━━━━━━━━ 图表规范 ━━━━━━━━━━

1. 连线规则（最重要）：
   - 必须使用 skinparam linetype ortho，所有连线为水平/垂直正交直线，禁止曲线
   - 实线箭头（-->）表示数据传递/调用
   - 虚线箭头（..>）表示日志记录
   - 每条连线必须标注接口名称（如：处理数据、识别结果、权限验证等）

2. 布局规则：
   - 模块按功能分层排列，从上到下依次为：数据采集层 → 核心数据层 → 业务处理层 → 展示应用层
   - 平台公共服务（接口服务、权限管理、日志管理）放在右侧纵向排列
   - 使用 skinparam nodesep 和 ranksep 控制间距，确保直线不过于密集交叉
   - 外部角色（用户/外部系统）使用 actor 关键字

3. 模块表示规则：
   - 普通功能模块用 rectangle，格式为"模块名\n(补充说明)"
   - 数据存储模块用 database
   - 每个模块必须定义英文别名（如 as data_storage）方便连线引用
   - 模块描述用中文，别名用英文下划线命名

4. 注释标注规则：
   - 对核心模块添加 note 注释，说明其关键职责或技术实现方式
   - 注释用 note ... as N1 定义，用 N1 .. 模块别名 关联

5. 样式规则：
   - 字体：Microsoft YaHei，字号 10
   - 背景色：#FFFFFF
   - 模块填充：#FFFFFF，边框：#000000，圆角 5
   - 注释填充：#FAFAFA，边框：#181818
   - 图例填充：#DDDDDD

6. 代码组织规则：
   - 用中文注释将代码分为清晰的段落：全局样式、角色定义、各层模块定义、注释标注、实线接口关系（按链路分组）、虚线日志关系、图例
   - 接口关系按功能链路分组并注释，如：外部访问链路、数据采集链路、业务分析链路等
   - 每组链路前用 ' --- XXX链路 --- 注释分隔

7. 图例：
   - 右下角添加 legend，说明实线和虚线的含义

━━━━━━━━━━ 我的系统信息 ━━━━━━━━━━

系统名称：【填写系统名称，如：飞行训练评估系统】

外部角色：
- 【如：外部系统/用户】

模块清单（按层级列出）：

第一层 - 数据采集与归档：
- 【模块名（补充说明），如：数据接入模块（批量上传/计划提取）】
- 【...】

第二层 - 核心数据与处理：
- 【如：数据存储管理模块（数据库+文件系统+归档）—— 用 database 表示】
- 【...】

第三层 - 业务分析：
- 【如：动作识别模块、质量评估模块、风险监测模块】
- 【...】

第四层 - 展示应用：
- 【如：三维复盘模块、辅助讲评模块、能力画像模块】
- 【...】

平台公共服务（右侧）：
- 【如：接口服务模块（WebAPI/SDK）、权限管理模块、系统日志管理模块】

注释标注：
- 【模块名】→ 注释内容【如：数据存储管理模块 → "核心数据中心\n集中存储管理"】
- 【...】

接口关系（实线 - 数据传递/调用）：
- 【源模块】→【目标模块】：【接口名称】
  例如：
  - 外部系统/用户 → 接口服务模块：数据查询/调用
  - 接口服务模块 → 权限管理模块：权限验证
  - 数据存储管理模块 → 动作识别模块：处理数据
  - 动作识别模块 → 数据存储管理模块：识别结果
  - 【...逐条列出所有实线接口...】

日志关系（虚线 - 日志记录）：
- 【源模块】..>【日志模块】：【日志类型】
  例如：
  - 数据接入模块 ..> 系统日志管理模块：操作日志
  - 动作识别模块 ..> 系统日志管理模块：识别日志
  - 【...逐条列出所有虚线日志...】

━━━━━━━━━━ 输出要求 ━━━━━━━━━━

请直接输出完整可运行的 PlantUML 代码（@startuml 到 @enduml），不要省略任何模块或接口。代码中用中文注释清晰分段，方便后续维护修改。
```

## 使用示例

假设你要画一个"智慧校园管理系统"的接口图，只需要把【】中的内容替换：

```
系统名称：智慧校园管理系统

外部角色：
- 学生/教师/管理员

模块清单：

第一层 - 数据采集：
- 考勤采集模块（人脸识别/刷卡）
- 成绩导入模块（批量导入/API对接）

第二层 - 核心数据：
- 数据中心（MySQL+Redis+文件存储）—— 用 database 表示
- 数据清洗模块（去重/标准化/校验）

第三层 - 业务分析：
- 学业分析模块
- 行为预警模块
- 课程推荐模块

第四层 - 展示应用：
- 学生画像模块
- 教学看板模块
- 家校互通模块

平台公共服务：
- API网关（REST/GraphQL）
- 统一认证模块（OAuth2.0）
- 审计日志模块

...后续填写接口关系和日志关系...
```

然后整段复制给 AI 即可生成完整的 PlantUML 代码。

---

## 注意事项

1. **模块数量建议**：单张图控制在 10-18 个模块内，太多会导致连线交叉严重
2. **接口关系要完整**：每条连线都要列出，不要让 AI 自己猜，否则可能遗漏或编造
3. **双向数据流分开写**：如 A→B：处理数据 和 B→A：识别结果，写成两条而不是双向箭头
4. **渲染工具推荐**：plantuml.com/plantuml（在线）、VS Code PlantUML 插件（本地）
5. **如果连线交叉严重**：可以调整 nodesep（水平间距）和 ranksep（垂直间距）的数值，或调整模块声明顺序来优化布局
